JSON- What is it?

From the official JSON site: JSON (JavaScript Object Notation) is a lightweight data-interchange format. It is easy for humans to read and write. It is easy for machines to parse and generate. It is based on a subset of the JavaScript Programming Language, Standard ECMA-262 3rd Edition - December 1999. JSON is a text format that is completely language independent but uses conventions that are familiar to programmers of the C-family of languages, including C, C++, C#, Java, JavaScript, Perl, Python, and many others. These properties make JSON an ideal data-interchange language.

JSON is built on two structures:

  • A collection of name/value pairs. In various languages, this is realized as an object, record, struct, dictionary, hash table, keyed list, or associative array.
  • An ordered list of values. In most languages, this is realized as an array, vector, list, or sequence.

Accessing the JSON object

The way we access the key-value pairs inside the JSON object is as follows.

See the Pen dvoRzd by Ivan Xa (@iax333) on CodePen.


The JSON object that is sent back to us is saved to "response" as a JSON object we can access via the line 

 var response = JSON.parse(req.responseText);

Then we access the object and its keys as 

response.copyright;
response.date;
response.explanation;
response.hdurl;
response.media_type;
response.service_version;
response.title;
response.url;

so on, so forth. You would have to save these calls to a variable if you want to use them later on in the code. If you don't know what "document.getElementById" does, or what "textContent" is, don't worry. We will be going over those in the next page.

What's important is that we can interpret JSON objects with Javascript. In the next page, we will be learning how to manipulate the DOM and use our JSON object to add data to the DOM.
